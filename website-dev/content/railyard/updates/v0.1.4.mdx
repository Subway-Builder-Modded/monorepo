<ChangelogSection type="features">

- Added a library page to easily view and manage installed content
- Installs can now be cancelled during the download phase
- Added support for optional GitHub token as a workaround for GitHub API limits

</ChangelogSection>

<ChangelogSection type="improvements">

- Added sorting by country/downloads/last updated (default is now last updated)
- All assets now show download counts on **Browse**, with maps also showing flag icons corresponding to the country of the map
- Search now only shows tags when an asset with that tag exists
- Separated toasts for download progress and download success and added an indicator for how many downloads are in queue
- Added an updated multistep loading screen on startup
- The app now launches in fullscreen
- Linux builds are now through Flatpak. This means you will need to have Flatpak installed in order for the app to function. In addition, this means the autoupdater will be broken for one update on Linux (you will need to manually install the flatpak included in the v0.1.4 release yourself)

:::note

You may not be able to find normal file paths when using the flatpak on linux. This is normal. Your host file system will be mounted on /run/host. You **will** have to update your paths when switching to the flatpak build.

:::

</ChangelogSection>

<ChangelogSection type="bugfixes">

- Fixed the error log copy button overlapping with text
- Removed scrolling left and right/zooming in and out
- Profiles will now automatically purge entries related to uninstallable content
- Resolved issue where multiple Railyard application instances could be started simultaneously
- Assets with no installable versions are hidden in **Browse** and individual versions that are not installable are hidden on the asset's page
- Fixed the in-game mod loader version (appeared as `vv0.1.3`)

</ChangelogSection>

<ChangelogSection type="notes">

- Changed app instance title from `railyard` to `Railyard`

</ChangelogSection>
