<ChangelogSection type="features">

- Added update indicators to the Library page if an asset's version is outdated
- Multiple assets can be updated at once via selection
- Added a theme switcher and 6 new themes: `Dark`, `Dark (Soft)`, `Dark (Contrast)`, `Light`, `Light (Soft)`, `Light (Contrast)`

</ChangelogSection>

<ChangelogSection type="improvements">

- Added corrupted/missing install-state recovery on startup
- Added validation checks for GitHub personal access tokens
- Game logs are now separated by session

</ChangelogSection>

<ChangelogSection type="bugfixes">

- Removed the leftover right-click context menu
- Fixed pinching (on a touchscreen device) still allowing you to zoom
- Fixed selecting multiple projects on the Library page not clearing selection when deleting via the trash can
- Fixed the paths used to install the security sandbox for Subway Builder
- Fixed the paths used to spawn the game

</ChangelogSection>

<ChangelogSection type="notes">

- Added deep link integration with the website. Clicking download for an asset on the website will now open Railyard and navigate to the asset's page
- Restored auto-update functionality on Linux
- Added additional logging to help with debugging
- Railyard now allows only one cancellation button to be present during install
- Standardized coloring between update/delete/install interfaces

</ChangelogSection>
