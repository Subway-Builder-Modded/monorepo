<ChangelogSection type="features">

- Railyard now attempts to auto-resolve the new Windows executable path for game version `1.4.x`
- Added sidebar filters to **Browse** for Compatible/Incompatible/Test content

<p align="center">
  <Image width="600" alt="image" src="/images/railyard/updates/v0.2.5/sidebar-filters.png" />
</p>

- Added support to import multiple assets at once
  - Each asset is reviewable independently, with clear conflict/validity messages

<p align="center">
  <Image width="800" alt="image" src="/images/railyard/updates/v0.2.5/review-import.png" />
</p>

- Unified incompatible asset handling across the application
  - Whenever an asset is incompatible with the detected game version, a tooltip will display describing the issue

<p align="center">
  <Image
    width="400"
    alt="image"
    src="/images/railyard/updates/v0.2.5/incompatibility-tooltip.png"
  />
</p>

</ChangelogSection>

<ChangelogSection type="improvements">

- Undetectable game version no longer causes Railyard to aggressively prune subscriptions
- **Discover** now reflects the same incompatible chips as **Library**/**Browse**
- Invalid imports now list which files are missing instead of saying a generic "missing one or more required files" message

</ChangelogSection>

<ChangelogSection type="bugfixes">

- Fixed an issue where the `Flatpak` mount for Linux version detection wouldn't launch due to location or would become orphaned due to improper process killing
- Fixed a bug with **Enable Dev Tools** on Linux by passing the variable into `flatpak-spawn`
- The install page no longer displays upgrades to asset versions incompatible with game version

</ChangelogSection>
