<ChangelogSection type="features">

### Demand

- Added a new prepare_osrm function to build a local OSRM server via Docker
- Added a new IGNORE_SCHEMA parameter to avoid loading/saving schema files
  - This may break adding special demand, but you can apply most methods without issue in this mode

</ChangelogSection>

<ChangelogSection type="improvements">

### MapGen

- Optimized ocean depth gridding
- All map files now strictly clip to the bounding box

### Demand

- Made `consolidate_pops` method more efficient and now ignores all special demand types
- `_load_schema` now allows users to specify an exponent dictionary to use instead of Depot's default

</ChangelogSection>

<ChangelogSection type="bugfixes">

### Demand

- Various bugfixes for `create_description`
- Fixed `scale_demand` to ignore all special demand types

</ChangelogSection>
